chain_interactions#

This module provides functions interacting with the chain for end-to-end testing; these are not present in btsdk but are required for e2e tests

async chain_interactions.wait_epoch(subtensor, netuid=1)#

Waits for the next epoch to start on a specific subnet.

Queries the tempo value from the Subtensor module and calculates the interval based on the tempo. Then waits for the next epoch to start by monitoring the current block number.

Raises:

Exception – If the tempo cannot be determined from the chain.

Parameters:

  • subtensor (bittensor.core.subtensor.Subtensor)

  • netuid (int)

async chain_interactions.wait_interval(tempo, subtensor, netuid=1, reporting_interval=10)#

Waits until the next tempo interval starts for a specific subnet.

Calculates the next tempo block start based on the current block number and the provided tempo, then enters a loop where it periodically checks the current block number until the next tempo interval starts.

Parameters:

  • tempo (int)

  • subtensor (bittensor.core.subtensor.Subtensor)

  • netuid (int)

  • reporting_interval (int)
